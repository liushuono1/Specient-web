# 客户登录与 Demo 系统设计

## 1. 页面定位
功能性页面，简洁、高效。提供不同行业的 Demo 入口。

## 2. 页面流程
1.  **登录页 (Login)**
2.  **仪表盘 (Dashboard)**
3.  **Demo 运行区 (Demo Sandbox)**

## 3. 详细设计

### A. 登录页 (Login)
*   **布局**: 左右分栏。左侧为品牌宣传图（动态的 Spec 网格），右侧为登录表单。
*   **表单**: 
    *   账号 / 密码
    *   企业邮箱验证码登录
    *   “申请试用账号”链接
*   **文案**: 欢迎回到 SpecAgent 智能控制台。

### B. 仪表盘 (Dashboard)
*   **布局**: 顶部为用户信息栏。主体为卡片式导航。
*   **卡片内容**: 
    *   **"我的行业 Demo"**: 根据账号属性（如是航空公司客户）高亮显示对应行业 Demo。
    *   **"通用能力广场"**: 展示文档生成、智能搜索等通用工具。
    *   **"API 密钥管理"**: 开发者选项。
*   **视觉风格**: 极简 SaaS 风格。白底黑字，浅灰色背景。

### C. Demo 运行区 (Interactive Sandbox)
*   这是一个通用的框架页面，根据选择的 Demo 加载不同的 iframe 或组件。
*   **结构**:
    *   **左侧栏**: 参数配置 (Spec Configuration)。用户可以在这里调整参数（如：公文类型=通知，紧急程度=特急）。
    *   **中间栏**: 交互对话框 (Chat Interface)。用户输入自然语言指令。
    *   **右侧栏**: 实时预览 (Live Preview)。显示生成的结果（文档、图表、行程单）。
    *   **底部**: 日志控制台 (Log Console)。实时滚动显示 Agent 的思维链 (Chain of Thought) 和 Spec 校验过程：“正在解析指令 -> 匹配 Spec 库 -> 校验通过 -> 执行操作”。

## 4. 技术与交互细节
*   **响应式**: 支持平板操作，方便销售人员现场演示。
*   **反馈机制**: 每次操作成功，右下角弹出绿色 Toast 提示“Spec Verified”。

## 5. 数据来源
*   本系统的静态配置文案由 `portal_data.json` 驱动。
